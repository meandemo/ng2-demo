HTML Code Used on each Section

Section A

This div is fixed at the top of the viewport. The code for this fixed div is: 

          <sticky-div  maxscroll>
          ...
          </sticky-div>

The presence of the property maxscroll without argument forces a fixed div at that location: all scrollable content pass under this div. You could use maxscroll=0 as an alternative syntax.

Section B

This div is fully scrollable, but we didn't use div, instead we used 

          <sticky-div #_inst0>
          ...
          </sticky-div>

Without maxscroll property, this component behaves like a normal div. Note the local variable &x23;_inst0, please read-on to see how it is used.

Section C

This navbar has a sticky behavior, we used the following code 

          <sticky-div [maxscroll]="_inst0.height()">
          ...
          </sticky-div>

We have use the local variable to obtain the height of the scrollable div just above. This height gives the scrollable amount for our sticky navbar using the one-way property binding syntax [maxscroll].

An alternate way, it to use a normal div for section B: 

          <div #_inst0>
          ...
          </div>

and use the clientHeight property to get the height of the div. So the code for section C could also be written: 

          <sticky-div [maxscroll]="_inst0.clientHeight">
          ...
          </sticky-div>

Section D

Same as Section B, this div is fully scrollable: 

          <sticky-div #_inst1>
          ...
          </sticky-div>

We have introduced a second local variable: #_inst1.

Section E

This navbar has a sticky behavior, we used the following code: 

          <sticky-div [maxscroll]="_inst0.height() + _inst1.height()">
          ...
          </sticky-div>

Two local variables are needed to compute the maxscroll property. It's the height of all the scrollable divs above the current one: i.e. section (B) & (D).

Section F, G and H

These sections illustrate a common error with sticky-div. The maxscroll value of section G is incorrect.
